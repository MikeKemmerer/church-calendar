# Contributing

## Reporting Issues

Open a GitHub issue with:
- A clear description of the problem or suggestion
- Steps to reproduce (if applicable)
- Browser/OS details for display issues

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b my-feature`)
3. Make your changes and test locally
4. Commit with a clear message
5. Open a pull request against `main`

## Development Notes

- **Languages:** Python 3, HTML/CSS/JavaScript
- **Target hardware:** Raspberry Pi Zero 2W — test on low-resource hardware when possible
- Python dependencies are listed in the project; install with `pip`
- Keep the frontend lightweight — no heavy frameworks or build tools
- Test the calendar display in a full-screen kiosk browser

## Code Style

- Python: follow PEP 8
- JavaScript/HTML/CSS: keep it simple and readable
- Avoid adding external dependencies unless absolutely necessary
